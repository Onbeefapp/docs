Introdução

Esta API segue o padrão Open Delivery e fornece recursos para:

  • Autenticação
  • Gerenciamento de merchants (informações, cardápios e status)
  • Cadastro e atualização de dados do merchant
  • Recuperação de pedidos e atualização de status
  • Eventos de pedidos e confirmações de entrega

Documentação Open Delivery: https://abrasel-nacional.github.io/docs

Autenticação

A Onbeef suporta apenas autenticação OAuth 2.

Obter Token

Solicita um novo token de acesso para acessar os recursos da API Onbeef.

Request Body schema: application/json
required
client_id
string

ID do merchant, gerado e obtido ao fazer login no App Onbeef na seção Integrações.

client_secret
string

Chave secreta do merchant, gerada no App Onbeef na seção Integrações.

grant_type
string
Value: "client_credentials"

Tipo de autenticação OAuth. Suportado: client_credentials.

Responses

Ambiente de Produção

Obter novos eventos

Authorizations:

query Parameters
eventType
Array of strings <= 15 items
Items Enum: "CREATED" "CONFIRMED" "PREPARING" "DISPATCHED" "READY_FOR_PICKUP" "PICKUP_AREA_ASSIGNED" "PICKED_UP" "DELIVERED" "CONCLUDED" "CANCELLATION_REQUESTED" "CANCELLATION_REQUEST_DENIED" "CANCELLED" "ORDER_CANCELLATION_REQUEST" "CANCELLED_DENIED"

Tipos de eventos desejados. Todos os outros eventos não selecionados pelo grupo serão automaticamente reconhecidos e omitidos da resposta. Deve ser omitido ou não vazio.

Definições de tipos de eventos:

CREATED: Pedido criado.
CONFIRMED: Pedido confirmado.
PREPARATION_REQUESTED: Informa ao Software Service que pode iniciar a preparação do pedido.
PREPARING: Preparação do pedido iniciada.
READY_FOR_PICKUP: Pedido pronto para retirada pelo cliente ou entregador.
DISPATCHED: Pedido saiu para entrega.
PICKUP_AREA_ASSIGNED: Informa ao Software Service a área designada para retirada do pedido.
PICKED_UP: Pedido foi retirado pelo cliente.
DELIVERED: Pedido entregue ao cliente.
CONCLUDED: Pedido concluído.
CANCELLATION_REQUESTED: Enviado pelo Software Service para o Ordering Application.
CANCELLATION_REQUEST_DENIED: Deve ser enviado do Ordering Application para o Software Service.
CANCELLED: Quando o pedido é efetivamente cancelado.
ORDER_CANCELLATION_REQUEST: Deve ser enviado do Ordering Application para o Software Service.
CANCELLED_DENIED: Quando o pedido de cancelamento do Ordering Application é negado pelo Software Service.
